   Features or Characteristics of JS
   1.Client-side scripting language
     (browser)    (no compiler)

     -> Javascript is run on the user's browser(chrome, edge, safari, firefox,etc.)
     -> Browser will download the JS files to execute.

   2.Dynamically Typed programming language

     -> There is no need to mention the datatype.
     -> The datatype will be assigned by the JS engine at the runtime of JS file.(the js code is directly run as there is no compiler present(no compilation)).

     let a=10;
     console.log(a);//10
               ^
               |
               number

   3.Weakly typed programming language.

     -> The javascript will change the datatype when it is needed(required).
     console.log(  5  +  "5"  );//55

                   V      V
                  num    string
                   |
                   V
                  "5"
   
   4.Interpreted programming language.

     -> code will execute line-by-line manner.
     (execution stops after meeting a bad line).

   5.Single Threaded programming language
       |          |
       v          v
       one  (A piece of code)

       -> A single threaded programming language means one task at a time.
       ->Single ->one
       -> Thread -> a piece of code

        __________       __________      __________
       |   task1  | ->  |   task2  | -> |   task3  |
       |__________|     |__________|    |__________|
       -> Javascript is a single threaded programming language because it is having only one *call stack*.
       -> Inside the call stack one task will be executed at a time. the other tasks wait to be executed in a queue.

   6.Synchronous in nature.

     -> By default javascript follows synchronous behaviour. 
     -> Synchronous means the code will be executed in a sequential format.
     -> One line, one task at a time until and unless the execution of the first line will not complete; the second line has to wait for execution.

     eg:
     <script>
     console.log("Task Started");
     alert("Task Paused");// Block the execution of the next line until and unless you click on the ok button.
     console.log("Task ended");
     </script>
     -> Blocking in nature

   7.Asynchronous in nature.

     ->We can make javascript asynchronous by using timing functions, promises, etc.
     ->We can perform multiple tasks at a time even without blocking the code

     -> non blocking in nature.

     eg:

     <script>
     console.log("Task Started");
     //Task will execute in the background without blocking the execution of next lines.
     setTimeout(()=>{
        alert("Task Paused");
     },5000);
     console.log("Task ended");
     </script>

   8.Object based programming language.
    
     -> Data will be stored in the form of object for memory allocation.
     -> It is also called as Object Based prototype.

   9.Event Driven

     -> Javascript supports multiple types of events such as mouse event, form events, keyboard events, etc.

   10.Run-on-both-side

     ->Javascript will be executed on client-side as well as on the server-side.

   11.Client-side-validation

     ->Javascript will validate the data on the brower side itself before sending to the server.

     eg: form data (if password and confirm_password are not matching, it will display an error notification on the browser side itself)

   12.Cross-platform
     -> js will execute of windows, mac, linux and mobile devices.
